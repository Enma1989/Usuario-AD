Criando Um Usuario AD

Guia passo a passo para Suporte N1

Antes de começar, leia com atenção.!

Criar um novo usuário no servidor AD é uma tarefa fácil, mas nem por isso, devemos relaxar e não tomar medidas de segurança e cuidados antes de começar. Para iniciar esta tarefa, devemos ter em mãos as seguintes informações:

  • Nome completo do Usuario que será criado
  • Informações importantes como departamento, a quem responde, etc.
  • Usuario que usaremos como "espelho"
  • Permissões adicionais se precisar
  • Senha de acesso

Entrando no servidor AD

Acessando ao Server Manager

Acessamos com nossas credenciais ao site do JUMP, colocando nosso usuário e senha, será feita uma verificação de duas etapas antes de acessar ao site. Ao finalizar colocamos na barra de pesquisa o nome da empresa que solicita a Criação do novo usuário; selecionamos WAD (geralmente aparece com esse nome). Dentro do servidor da empresa solicitante vamos ao menu de Windows Server, e pesquisamos: "Server Manager" e apertamos entre. Selecionamos nos resultados "Server Manager" Abrindo assim a tela que aparece na imagem ao inicio.

Active Directory Users and Computers

Acesso ao Active Directory

Na tela do Server Manager vamos para a barra de opções e procuramos "Tools", fazemos click no botão, aparecendo assim uma lista de opções como na imagem ao lado. Na barra de opções procuramos a opção: Active Directory users and Computers, fazemos click e aguardamos a janela abrir.

Active Directory Users and Computers II

Pastas do Active Directory

Na tela do Active Directory encontramos as pastas que contém os dados com as informações dos usuários e departamentos e outras informações. Cada banco de dados tem pequenas diferenças que é bom conhecer de cliente em cliente; na lista de pasta na esquerda encontraremos a pasta que contém os departamentos da empresa, selecionamos para ver em qual departamento estará o usuário que usaremos como espelho (deverá ser o mesmo departamento onde trabalhara o novo usuário).

Active Directory Users and Computers II

Pastas do Active Directory

Fazemos click na pasta onde encontramos ao nosso usuário espelho...

Active Directory Users and Computers II

Aba Criação de Usuario

Fazemos click direito acima do nome de nosso usuário espelho, logo, dentro da lista, selecionamos Copiar.

Active Directory Users and Computers II

Aba Criação de Usuario

Passamos a preencher os campos da janela "Copy Object - User". Começamos Colocando o primeiro nome do usuário, podemos deixar o campo "initials" (Iniciais) em branco; seguidamente colocamos sobrenome, à medida que Preenchemos os dados, o nome completo do usuário aparecera no campo "Full name". No campo "user logon name" colocamos o nome de usuário que usará o novo funcionário; falando disso, muitos clientes já usam um formato de nome de usuário que devemos respeitar, toma muito cuidado e atenção no chamado de criação de como deverá colocar em este campo. automaticamente aparecerá no campo "User logon name (pre-Windows 200)" o nome de usuário que acabamos de criar. ele ficará como a imagen ao lado.

Active Directory Users and Computers II

Aba Criação de Senha

Neste campo, colocamos a senha fornecida pelo solicitante, ou, colocamos a senha criada por nós (sempre seguindo as normas e procedimentos estabelecidos pelo departamento de Ciber Segurança da Gate7). A universidade Federal do Recôncavo da Bahia, dá umas dicas que podemos encontrar no site deles, no apartado: CyberSecurity: Utilize senhas com 8 ou mais caracteres, misture letras em maiúsculo e minúsculo, misture letras, números e símbolos como (! @ # $ % & *), use a semelhança de fonética como ("ca" e "k") ou visual como ("w" e "vv") (3=E) (5=S) (1 = L / j = ! ou i) (8=B) (S / $ / 5) (A=@) (W=M) (0 / O / *).

Active Directory Users and Computers II

Aba Criação de Senha

Também encontraremos uma série de opções: “User must change password at next logon” significa que o servidor solicitará mudar a senha para o novo usuário na primeira vez que entre na conta do computador. “user cannot change the password” significa que o usuário não poderá alterar a senha pelos meios fornecidos por Windows. “Password never expires” significa que a senha nunca expira. “Account is disabled” usamos esta opção para desativar a conta do usuario.

Conferindo as informações

Nesta tela temos a confirmação do que foi feito nos passos anteriores, por favor, confira bem todos os detalhes antes de finalizar o processo de criação de usuário, se assegure que o nome esteja escrito conforme no chamado, igualmente o nome de usuário e a opção que foi colocada para a senha, seja que ela possa ser alterada no primeiro logon ou, que o usuário não possa alterara-la.

Parabéns, você criou um novo usuário no AD Server!
